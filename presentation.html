The Problem

01 / Context

What we inherited

  • Spring Boot Java monolith — legacy/
  • AngularJS 1.2.16 frontend (2013-era)
  • 3 DB backends: JPA/H2, MongoDB, Redis
  • 29 seed albums, no tests, no docs
  • REST API with undocumented quirks

The risk

  • Undocumented behavior = no safety net
  • One wrong change breaks production
  • Big-bang rewrite has 70% failure rate

What we found first

8 undocumented behavioral quirks before we wrote a single line of new code:

  • LB-1 PUT creates, POST updates — verbs swapped
  • LB-2 UI only ever calls POST
  • LB-3 Two identity fields: id + ghost albumId
  • LB-4 No server-side filtering — all client-side
  • LB-5 trackCount in model, absent from UI
  • LB-6 Default sort is "name" — field doesn't exist
  • LB-7 GET /albums/{id} exists, UI never calls it
  • LB-8 fill-heap endpoint hidden from UI

Our Approach

02 / Strategy

Strangler Fig

Extract services incrementally alongside the running monolith. Legacy stays live. New service takes over one domain at a time. No cutover risk until we're ready.

Pin Before You Cut

Write characterization tests that pin existing behavior — bugs and all — before any extraction. The tests are our safety net, not our wishlist.

The Fence

An explicit anti-corruption layer prevents legacy field shapes (albumId, trackCount) from leaking into the new service's public API.

Phase 1 — Now Phase 2 — Next Phase 3 — Future Browser → Legacy monolith Browser → Proxy/Facade Proxy → Album Catalog only ↓ ↓ ↓ Legacy album endpoints retired /albums (UI) /albums (UI) /api/albums Album Catalog (FastAPI ✓)

Claude Code Patterns

06 / How We Used Claude

Configuration — the Write Guard Hook

.claude/settings.json PreToolUse hook — blocks writes to legacy/ unless CLAUDE_LEGACY_WRITE_ALLOWED=1 "If characterization tests exist, legacy/ is read-only. Set the env var to override intentionally." → Claude cannot accidentally mutate the patient while writing the extraction.

Agentic Parallelisation

Waypoints 3, 4, and 5 ran as three independent agents in a single fan-out — then Waypoint 6 as a fourth:

Coordinator ├── Agent A → Waypoint 3: The Map (ADR) ├── Agent B → Waypoint 4: The Pin (tests) ← parallel └── Agent C → Waypoint 5: The Cut (service) # After all three complete: └── Agent D → Waypoint 6: The Fence (ACL) Wall-clock ≈ slowest single agent, not sum of all three.

Each agent received its full scope explicitly — subagents don't inherit coordinator context (per CLAUDE.md).

Context via ADRs

Two ADRs encode decisions as durable context — future agents and engineers don't re-derive what was already decided.

Explore Subagent

Read-only Explore agent mapped legacy/ before any changes were proposed — protects main context from noise.

CLAUDE.md as System Prompt

Project rules, directory layout, and subagent guidance live in CLAUDE.md — every agent session starts with the same invariants.
